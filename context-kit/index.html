The Problem

Claude reads everything.
Then runs out of room.

When you point Claude at a folder with dozens of files, it tries to read them all. It burns through its context window on irrelevant content, then has no room left for actual work. Output quality drops. Sessions stall.

claude — cowork session
→ Reading project folder… (147 files)
  ├── archive/quarterly-report-2023.docx
  ├── drafts/old-brand-guide-v1.pdf
  ├── drafts/old-brand-guide-v2.pdf
  ├── meeting-notes-jan-14.md
  … 142 more files
⚠ Context usage: 92%
✕ Insufficient context for task. Output truncated.
The Fix

Three components.
One protocol.

Each works on its own, but they're designed to work together. Start with one, add the rest when you're ready.

1

Global Instructions

Persistent rules you paste into Cowork's global settings. They tell Claude to apply the tiered access model to every folder — read canonical docs first, load domain files only when tasks need them, ignore archival content.

📄 global-instructions.md
2

Folder Manifests

A _MANIFEST.md file you drop into any working folder. It tells Claude which documents are source-of-truth, which subfolders map to which domains, and what to skip entirely.

📄 manifest-template.md
3

Navigator Skill

A Cowork skill that enforces the orientation protocol — read manifest, load Tier 1, assess relevance, load Tier 2 selectively. Handles conflict resolution and scopes sub-agents automatically.

📄 SKILL.md
The Tier Model

Not all files are
created equal.

The system classifies every file in a folder into one of three tiers. Claude reads them in order of priority — most important first, irrelevant never.

TIER 1 — CANONICAL
Source-of-truth documents
Manifests, master references, approved final versions. The ground truth for the folder.
Always loaded first, before any work begins. Takes precedence over all other files if information conflicts.
TIER 2 — DOMAIN
Working files in domain subfolders
Brand docs, pricing models, engineering specs — read only when the task touches that domain.
Loaded selectively, only when the current task requires files from that specific domain.
TIER 3 — ARCHIVAL
Old versions, drafts, superseded files
Files in archive/, old/, drafts/, or anything dated older than 6 months.
Never loaded unless you explicitly ask Claude to read them. Completely ignored by default.

global-instructions.md SETTINGS
## Context Management — Tiered File Access
 
### Tier 1 — Canonical (Always Read First)
Before doing any work, scan the folder for
a `_MANIFEST.md` file or a `_canonical/`
subfolder. Read these first to orient.
 
### Tier 2 — Domain-Relevant
Only read files in subfolders that are
directly relevant to the current task.
 
### Tier 3 — Archival
Do not read unless explicitly asked.
_MANIFEST.md FOLDER
## Canonical Documents (Tier 1)
 
| Document | Covers |
|----------|--------|
| brand-guide-v3.docx | Brand identity |
| pricing-model.xlsx | Current pricing |
 
## Domain References (Tier 2)
| pricing/ | Read for pricing tasks |
| brand/ | Read for content tasks |
 
## Archival (Tier 3 — Ignore)
| old/ | Superseded by above |
Getting Started

Start small.
Add layers as needed.

You don't need everything at once. Each step independently improves Claude's behavior.

🟢
Start Here
Paste the Global Instructions
Claude Desktop → Settings → Cowork → Edit Global Instructions → paste global-instructions.md. This alone changes Claude's behavior — it stops reading everything and orients from folder structure first.
🔵
Next
Add a Manifest to Your Busiest Folder
Copy manifest-template.md into any folder, rename to _MANIFEST.md, fill it in. See if output quality improves. If it does, add manifests to other folders.
🟣
Later
Install the Navigator Skill
Zip the tiered-context-navigator/ folder, upload as a Cowork skill. This gives you consistent enforcement, especially for complex tasks with sub-agents.
